What is shell scripting?

A shell script is a computer program that is runned through the unix shell. Functions that can be performed by shell scripts are; file manipulation, printing text, and program execution. In shell script there is something called "scripting langauge"; Which is a programming language that supports scripts: programs written for a special environment that automates a execution of tasks that could alternatively be executed one-by-one by a human. A shell script can provide a variation of a system command where command options, or post-processing apply automatically, this allows the new script to still act as a fully normal Unix command. One of such scripting languages is GrepSedAwk, which is what I will be trying to teach you how to use these commands.

Grep

The Grep in "GrepSedAwk" stands for (Globally search a Regular Expression and Print), this section discusses GREP.

Basic Commands

In grep some of the most basic commands anyone can use and understand are:
- `grep -e` ~ is a command that tells computer to initiate the grep program with extended regular expressions; which means, - `grep -i` ~ the `grep -i` command tells the computer to run the grep program and forces grep to not allow the capitalization to matter when looking for expressions; which means, - `grep -v` ~ tells the program to return all the expressions that are listed that do not match the expression you typed in. - `grep -w` ~ this command only selects only the matches that are whole words. - `grep -c` ~ is a command that prints a count of lines that match the expression that was typed. A cool thing about the `grep -c` function is that, it can also be combined with the `grep -v` function to print a count of the lines that dont match. - `grep -l` ~ the `grep -l` command prints the name of every file that contains the expression you sent the program to look for. - `grep -n` ~ prints the line number before each of the lines that matches the expression typed into the program. - `grep -r` ~ the `grep -r`command reads all given files in all directories ad sub dierectories

Importance of Grep

Grep is a command used to search plain lines of code for expressions that was given to the program to search for. Its name comes from the ed command g/re/p (globally search a regular expression and print), which has the same effect: doing a global search with the regular expression and printing all matching lines. Originally Grep's creator Ken Thompson intended for grep to be used for linux but, was later developed to be usable for all systems similar to linux.

Sed

The Sed in "GrepSedAwk" stands for (Stream Editor), this section discusses SED.

Basic Commands

- `c` ~ the "c" command stands for change and when it's used it is to replace the line you are currently on with a new line. - `=` ~ the "=" command prints the you are currently on. - `r` ~ the "r *filename*" command reads the file - `i` ~ the "i" command inserts a new line before the oeiginal line - `d` ~ stands for delete, goes in front of the command that deals with removing stuff, whether it be lines or words. - `a` ~ stands for ammend, allows for words and phrases to be added after a sentence or a line of code.

Importance of Sed

Sed is a Unix utility that parses and transforms text, using a simple, compact programming language. Sed was developed from 1973 to 1974 by Lee E. McMahon of Bell Labs, and is available today for most operating systems. Sed allows one to change and edit the context and outputs of certain files without having to open any of those files.

Awk

The Awk in "GrepSedAwk" stands for (Aho, Weinberger and Kernighan), this section discusses AWK. In Awk There are three variations of AWK: - AWK~ the original from AT&T - NAWK~ A newer, improved version from AT&T - GAWK~ The Free Software foundation's version

AWK can be used for simple calculations; If you wanted to calculate without reading any lines for input, you could use the `BEGIN` keyword, combined with a `exit` command:

`BEGIN {`
`i=1;`
`while (i <= 10) {`
`print "The square of ", i, " is ", i*i;`
`i = i+1;`
`}`
`for (i=1; i <= 10; i++) {`
`print "The square of ", i, " is ", i*i;`
`}`
`exit;`
`}`

Awk Commands

- `~` this symbol means that the the expression matches what it was being compaired too. - When `~` has an `!` in front like `!~` this means that the expression doesn't match its comparison - `+` this symbol does addition when it is paired between two or more numbers. - `-` this symbol does subtraction when it is paired between two or more numbers. - `*` this symbol does multiplication when it is paired between two or more numbers. - `/` this symbol does division when it is paired between two or more numbers. - `%` this symbol does modulo when it is paired between two or more numbers. - ` ` When a space is placed it concatenates two strings to combine them together.

Importance of Awk

Awk is an excellent filter and report writer. Many UNIX utilities generates rows and columns of information. AWK is good for processing these rows and columns, and it is easier to use AWK than most programming languages. AWK does not only have string manipulation functions, so it can search for particular strings and modify its output, but also has associative arrays, which is something most computing languages are lacking. Awk programs follow the form `pattern {action}`, two important patterns are specified by the keywords "BEGIN" and "END".
